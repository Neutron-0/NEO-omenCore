# OmenCore v3.6.1 Remediation TODO

## Scope
Emergency stabilization and architecture remediation for v3.6.0 regressions, prioritizing deterministic fan/performance state sync, safe EC behavior, reliable hotkeys, and accurate OSD/tray telemetry.

## Current Program Mode
- Release-candidate preparation: active.
- v3.6.1 is now scoped as a focused stabilization release for the v3.6.0 regression baseline.
- Remaining non-blocking architecture consolidation is deferred to post-3.6.1 unless validation finds a hard release blocker.

## Post-3.6.1 Architecture Reduction Objective
- Move from "many semi-independent synchronized systems" to "one authoritative runtime state engine + read-only subscribers".
- Reduce runtime fragmentation by cutting state owners, polling loops, timers, orchestration layers, and duplicated refresh paths.
- Make runtime behavior deterministic under concurrency (tray/hotkey/automation overlap, minimized/background, resume).
