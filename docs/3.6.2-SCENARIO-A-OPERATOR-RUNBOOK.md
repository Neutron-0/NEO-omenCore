# OmenCore v3.6.2 Scenario A Operator Runbook

**Scenario A**: Screen-off idle + custom fan preset transitions  
**Goal**: Detect low-temp thermal-limit saturation, CPU package power lock, and custom-preset instability in one pass  
**Estimated Duration**: 35-45 minutes  
**Platform**: Windows (primary), laptop target hardware

---

## 1. Pre-Run Setup (5 minutes)

1. Reboot the machine and wait 2 minutes at desktop idle.
2. Launch OmenCore and verify telemetry is updating.
3. Open HWiNFO sensors and pin these fields:
   - CPU package temperature
   - Thermal throttling / thermal limit reason
   - CPU package power
   - Effective clock
   - CPU utilization
4. Ensure Windows sleep is disabled temporarily so screen-off can be tested without suspend.
5. Record baseline:
   - Ambient room estimate
   - Current fan preset
   - Current performance mode

---

## 2. Artifact Naming Convention

Use one run ID for all files, e.g. `A-2026-05-14-01`.

- HWiNFO screenshot: `A-2026-05-14-01-hwinfo.png`
- HWiNFO log (if available): `A-2026-05-14-01-hwinfo.csv`
- OmenCore diagnostics export folder: `A-2026-05-14-01-diagnostics`
- Operator notes: `A-2026-05-14-01-notes.md`

---

## 3. Execution Steps (One Pass)

### Step A1: Focused Idle Baseline (3 minutes)

1. Keep OmenCore visible and do not interact.
2. Observe and note CPU/GPU usage and package power trend.
3. Export diagnostics once with scenario label `scenario-a-focused-idle`.

Pass target:
- No thermal-limit reason saturation at low temperature.
- CPU package power behaves normally for idle state.

### Step A2: Tray Idle Baseline (3 minutes)

1. Minimize OmenCore to tray.
2. Wait 3 minutes.
3. Export diagnostics with scenario label `scenario-a-tray-idle`.

Pass target:
- No fan instability.
- No unexpected power lock.

### Step A3: Screen-Off Idle (10 minutes)

1. With OmenCore still running, turn screen off (no sleep).
2. Keep the system in this state for 10 minutes.
3. Turn screen on and immediately capture HWiNFO screenshot.
4. Export diagnostics with scenario label `scenario-a-screen-off-idle`.

Pass target:
- Thermal limit is not pegged at 100% with temperatures near 40-50C.
- No persistent package power cap around 20-30W after wake.

### Step A4: Custom Preset Transition Stress (15 minutes)

1. Apply custom fan preset.
2. Wait 90 seconds.
3. Switch to Auto.
4. Wait 90 seconds.
5. Switch back to custom preset.
6. Repeat this cycle 5 times.
7. During cycle 3 and cycle 5, capture HWiNFO screenshot.
8. Export diagnostics with scenario label `scenario-a-custom-preset-stress`.

Pass target:
- No spontaneous fan stop.
- No preset reset.
- No low-temp thermal-limit saturation.
- No persistent package power lock.

### Step A5: Persistence Check (5 minutes)

1. Keep custom preset selected.
2. Close OmenCore.
3. Re-open OmenCore.
4. Verify preset, fan behavior, and telemetry remain consistent.
5. Export diagnostics with scenario label `scenario-a-persistence-check`.

Pass target:
- Preset persists correctly.
- No reset to unintended state.

---

## 4. Failure Signature Rules

Mark Scenario A as **FAILED** if any of the following occur:

1. Thermal-limit reason is 100% while CPU package temperature is stably around 40-50C.
2. CPU package power remains abnormally capped around 20-30W after preset reapply.
3. Fan stops unexpectedly, or custom preset is reset without operator action.
4. Issue reproduces across app restart within the same run.

---

## 5. Immediate Recovery Actions

If failure signature appears:

1. Reapply current preset.
2. Switch to Auto, wait 30 seconds, then switch back to custom.
3. If not cleared, restart OmenCore and re-check.
4. If still not cleared, reboot and re-check quickly.

Record at which step the issue clears (if at all).

## 7. Exit Criteria

Scenario A is **PASS** only if:

1. No low-temp thermal-limit saturation appears.
2. No unexplained 20-30W package power lock persists.
3. No fan shutdown/reset behavior appears under custom preset stress.
4. Preset persistence survives app restart.

Otherwise classify as **FAIL** and escalate with full evidence bundle.
