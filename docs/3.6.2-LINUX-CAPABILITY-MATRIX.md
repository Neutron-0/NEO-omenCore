# OmenCore 3.6.2 Linux Capability Matrix

## Capability Classes
- `full-control`: writable fan + profile interfaces available.
- `profile-only`: platform profile writable, manual fan target unavailable.
- `telemetry-only`: sensors readable but writable controls unavailable.
- `unsupported-control`: no supported control path detected.

## Detection Inputs
- EC path: `/sys/kernel/debug/ec/ec0/io`
- hp-wmi root: `/sys/devices/platform/hp-wmi`
- ACPI profile: `/sys/firmware/acpi/platform_profile`
- hp-wmi profile variants:
  - `thermal_profile`, `thermal-profile`
  - `platform_profile`, `platform-profile`
- hwmon variants under hp-wmi:
  - `fan1_target`, `fan2_target`
  - `pwm1_enable`, `pwm2_enable`

## GUI/CLI Behavior
- Fan navigation visibility follows `SupportsFanSurface`.
- Manual fan actions still require writable fan target/output interfaces.
- Performance mode actions use profile interfaces first, optional `powerprofilesctl` fallback.
- Keyboard brightness operations report explicit unsupported reasons when `brightness` sysfs file is absent.
