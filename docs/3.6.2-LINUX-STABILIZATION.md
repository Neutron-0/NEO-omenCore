# OmenCore 3.6.2 Linux Stabilization

## Scope
- Added schema-versioned, migration-tolerant TOML loading in the Linux config layer.
- Unified Linux CLI config command on the authoritative TOML model.
- Centralized Linux sysfs path normalization for hp-wmi and ACPI profile discovery.
- Improved unsupported-feature messaging for performance and keyboard brightness paths.
- Hardened desktop integration launch behavior with multi-launcher fallback.
- Expanded diagnostics with config schema/load migration reporting.

## Key Outcomes
- Daemon/CLI config loading now tolerates deprecated and legacy key variants instead of hard-failing.
- Legacy keys like `perf.mode`, `startup.apply`, and dotted top-level aliases are migrated during load.
- Effective schema version is normalized to `2` on load/save.
- GUI fan navigation visibility now derives from runtime capability surface, not OS heuristic.
- The final Linux battery profile persistence path now uses the same TOML authority as the rest of Linux config.

## Additional Consolidation In The Runtime Recovery Pass
- Added `[battery] profile` to the authoritative TOML model in [src/OmenCore.Linux/Config/OmenCoreConfig.cs](../src/OmenCore.Linux/Config/OmenCoreConfig.cs).
- Migrated legacy `battery.profile` aliases into the TOML-backed battery section during tolerant load.
- Removed the remaining JSON `ConfigManager` helper from [src/OmenCore.Linux/Program.cs](../src/OmenCore.Linux/Program.cs), eliminating split ownership between TOML and ad-hoc JSON.
