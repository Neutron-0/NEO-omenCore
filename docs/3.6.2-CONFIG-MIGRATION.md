# OmenCore 3.6.2 Config Migration

## Schema
- Current schema version: `2`
- File key: `schema_version`

## Loader Behavior
- Parses TOML using table-model flow and applies known keys only.
- Unknown keys are tolerated and do not block startup.
- Loads system then user config with user-value override semantics.
- Sanitizes loaded values to safe ranges and known enums.

## Legacy Mapping
The loader maps old keys to current schema fields when encountered:
- `perf.mode` -> `performance.mode`
- `startup.apply` -> `startup.apply_on_boot`
- `keyboard.brightness_percent` -> `keyboard.brightness`
- `fan.profile` -> `[fan].profile`
- `fan.boost` -> `[fan].boost`
- `keyboard.color` -> `[keyboard].color`
- `general.polling_interval_ms` -> `[general].poll_interval_ms`

## Diagnostics
`OmenCoreConfig.LastLoadReport` tracks:
- loaded config paths
- detected schema versions per file
- applied migration entries
- parse/migration warnings

`omencore-cli diagnose` now includes schema version and migration warning context in output payloads.
