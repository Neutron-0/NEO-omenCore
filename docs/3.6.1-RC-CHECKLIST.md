# OmenCore v3.6.1 Release Candidate Checklist

## Release Intent
v3.6.1 is a focused stabilization release for the v3.6.0 regression baseline. It is not a feature release and it is not a claim of full runtime architecture consolidation.

Primary release goals:
- Stabilize fan/performance state convergence across UI, tray, hotkeys, and OSD.
- Reduce WMI fan-control CPU/write pressure during custom curve and hold sessions.
- Improve EC operation serialization across fan, performance, keyboard, power, diagnostics, and GPU boost fallback paths.
- Make unsupported or unverified capability states fail clearly instead of ambiguously.
- Preserve 3.2.5 fan/performance fundamentals and 2.9.0 tray no-freeze behavior.
- Ensure custom/manual fan curves publish only canonical runtime `Custom` state, with no ghost `Manual` / `Custom (Applied)` labels or duplicate OSD mode rows.

## Tester Validation Checklist

Ask testers to include model, ProductId, BIOS version, Windows/Linux version, install type, and whether OMEN Gaming Hub is installed/running.

Windows tester pass:
- [ ] Launch as Administrator and confirm version shows `3.6.1`.
- [ ] Switch fan modes from the main UI: Auto, Quiet, Performance/Extreme, Max.
- [ ] Use tray fan mode actions and confirm tray/header/checkmarks converge.
- [ ] Use hotkeys: fan cycle, performance cycle, Max Fan (`Ctrl+Shift+M`), boost, quiet.
- [ ] Toggle linked fan/performance mode and confirm sidebar, fan page, tray, and OSD agree after each change.
- [ ] Apply a custom fan curve and watch CPU usage for 10 minutes; report if OmenCore remains unusually high.
- [ ] Minimize to tray for 10 minutes, then restore; confirm telemetry, tray state, and OSD labels are fresh.
- [ ] Suspend/resume once if comfortable; confirm monitoring and fan control recover.
- [ ] Export diagnostics after testing and attach logs for any mismatch.

Linux tester pass:
- [ ] Launch CLI: `sudo ./omencore-cli status`.
- [ ] Launch GUI: `./omencore-gui`.
- [ ] Confirm profile controls are enabled only when supported.
- [ ] Try Refresh Sensors and confirm visible feedback changes.
- [ ] If profile controls are unavailable, run diagnostics and include hp-wmi/sysfs details.
- [ ] For newer OMEN boards with missing profile controls, test documented `hp_wmi.force_multiplex=1` guidance only if comfortable reverting kernel parameters.

Report immediately if:
- Fan mode or performance mode UI shows success but hardware state obviously does not change.
- Max Fan cannot be applied or cannot be exited.
- OmenCore sustains high CPU while idle or tray-only.
- OSD/tray/sidebar disagree after waiting several seconds.
- Diagnostics show repeated EC/WMI failures during normal mode changes.
